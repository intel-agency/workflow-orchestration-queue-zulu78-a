## Key Components

### Notifier Service (`src/notifier_service.py`)
- FastAPI application with `/webhooks/github` and `/health` endpoints
- HMAC SHA256 signature verification on webhook payloads
- Maps GitHub issue events to `WorkItem` and queues via `GitHubQueue`
- Entry point: `uv run uvicorn src.notifier_service:app --reload`

### Sentinel Orchestrator (`src/orchestrator_sentinel.py`)
- Long-running async polling loop for `agent:queued` issues
- Distributed locking via assign-then-verify pattern
- Heartbeat comments during task execution
- Jittered exponential backoff on rate limits
- Entry point: `uv run python -m src.orchestrator_sentinel`

### Work Item Model (`src/models/work_item.py`)
- `TaskType` enum: PLAN, IMPLEMENT, BUGFIX
- `WorkItemStatus` enum: maps to GitHub labels (agent:queued, agent:in-progress, etc.)
- `WorkItem` Pydantic model: unified data structure
- `scrub_secrets()`: regex-based credential redaction

### GitHub Queue (`src/queue/github_queue.py`)
- `ITaskQueue` ABC for future provider swapping
- `GitHubQueue` concrete implementation with HTTPX connection pooling
- Methods: add_to_queue, fetch_queued_tasks, update_status, claim_task, post_heartbeat

## CI Pipeline

The `validate.yml` workflow runs on push to `main` and on PRs:
1. **lint** — markdownlint, ruff, hadolint, yamllint
2. **scan** — gitleaks secret scanning
3. **test** — Pester tests + shell tests
4. **test-devcontainer-build** — devcontainer build verification

All GitHub Actions are pinned to specific commit SHAs with semver comments.

## Purpose

OS-APOW transforms GitHub Issues into automated execution orders for specialized AI agents. It is a headless system that moves AI from a passive co-pilot role to an autonomous background production service.

## Architecture

### Four Pillars

1. **The Ear (Notifier)** - FastAPI webhook receiver with HMAC validation
2. **The State (Queue)** - GitHub Issues as state machine with labels
3. **The Brain (Sentinel)** - Python async polling and task dispatch
4. **The Hands (Worker)** - DevContainer + opencode CLI for execution

## Configuration

All configuration via environment variables with `OS_APOW_` prefix:

- `OS_APOW_GITHUB_TOKEN` - GitHub authentication (required)
- `OS_APOW_GITHUB_ORG` - Organization name
- `OS_APOW_GITHUB_REPO` - Target repository
- `OS_APOW_WEBHOOK_SECRET` - HMAC secret for webhooks
- `OS_APOW_SENTINEL_ID` - Unique sentinel identifier
- `OS_APOW_LOG_LEVEL` - Logging level (default: INFO)
- `OS_APOW_LOG_FORMAT` - Log format (json/text)

## Notes for AI Agents
